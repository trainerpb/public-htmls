Brute-Force Methodologies

Brute-force relies on sheer computing power to evaluate every single possible configuration. It guarantees a correct answer simply by leaving no stone unturned.

Description: Checking every element in an array sequentially until the target is found. It makes no assumptions about the data being sorted.

Time: $O(n)$ Space: $O(1)$
Java: Linear Search
public int linearSearch(int[] arr, int target) {
    for (int i = 0; i < arr.length; i++) {
        if (arr[i] == target) return i;
    }
    return -1;
}

Description: Sliding a "pattern" window over a "text" one character at a time. If a mismatch occurs, it abandons all progress, shifts the pattern exactly one space to the right, and restarts.

Time: $O(n \times m)$ Space: $O(1)$
Java: Naive String Match
public void search(String txt, String pat) {
    int n = txt.length(), m = pat.length();
    for (int i = 0; i <= n - m; i++) {
        int j;
        for (j = 0; j < m; j++) {
            if (txt.charAt(i + j) != pat.charAt(j)) {
                break; // Mismatch! Reset j.
            }
        }
        if (j == m) System.out.println("Match at " + i);
    }
}

Description: Generating every possible subset of an array to see if elements sum to a target. It explores "include" and "exclude" branches for every item.

Time: $O(2^n)$ Space: $O(n)$
Java: Recursive Subset Sum
public boolean isSubsetSum(int[] set, int n, int sum) {
    if (sum == 0) return true;
    if (n == 0) return false;

    if (set[n - 1] > sum)
        return isSubsetSum(set, n - 1, sum);

    // Check inclusion branch OR exclusion branch
    return isSubsetSum(set, n - 1, sum) || 
           isSubsetSum(set, n - 1, sum - set[n - 1]);
}

Greedy Paradigm

Greedy makes the best immediate choice at the current moment. "Take the biggest bite possible, and never look back."

Description: Select the largest denomination that fits the remaining balance. Works perfectly for standard currencies but fails on arbitrary coin systems.

Time: $O(n \log n)$ Space: $O(1)$
Java: Greedy Coin Change
public void greedyChange(int[] coins, int amount) {
    // Assumption: coins array is sorted descending
    for (int coin : coins) {
        while (amount >= coin) {
            amount -= coin;
            System.out.print(coin + " ");
        }
    }
}

Description: You can break items into fractions. Sort items by value-to-weight ratio and take the highest ratios first until the bag is full.

Time: $O(n \log n)$ Space: $O(1)$
Java: Fractional Knapsack
public double getMaxValue(Item[] arr, int capacity) {
    // Sort arr by (value/weight) descending
    double totalValue = 0;
    for (Item i : arr) {
        if (capacity - i.weight >= 0) {
            capacity -= i.weight;
            totalValue += i.value;
        } else {
            double fraction = (double)capacity / i.weight;
            totalValue += (i.value * fraction);
            break; // Full
        }
    }
    return totalValue;
}

Java: Activity Selection
// Arrays start/finish sorted by finish time
public void printMaxActivities(int[] s, int[] f, int n) {
    int i = 0;
    System.out.print(i + " "); 
    for (int j = 1; j < n; j++) {
        if (s[j] >= f[i]) {
            System.out.print(j + " ");
            i = j; 
        }
    }
}

Dynamic Programming (DP)

DP fixes Greedy's blind spots. It builds an array (a "notepad") to store the optimal answer to every smaller sub-problem, guaranteeing the perfect global solution.

Description: Makes change for arbitrary coins [1, 4, 5]. Checks all previous array states to definitively find the minimum coins required.

Time: $O(n \times \text{tgt})$ Space: $O(\text{tgt})$
Java: DP Bottom-Up
public int dpCoinChange(int[] coins, int target) {
    int[] dp = new int[target + 1];
    Arrays.fill(dp, target + 1);
    dp[0] = 0; 
    for (int i = 1; i <= target; i++) {
        for (int c : coins) {
            if (i >= c) dp[i] = Math.min(dp[i], 1 + dp[i - c]);
        }
    }
    return dp[target];
}

Description: Finding distinct ways to climb $N$ steps by summing the ways to reach the previous two steps. Eliminates exponential recursive branching.

Time: $O(N)$ Space: $O(N)$
Java: Climbing Stairs DP
public int climbStairs(int n) {
    if (n == 1) return 1;
    int[] dp = new int[n + 1];
    dp[1] = 1; dp[2] = 2; 

    for (int i = 3; i <= n; i++) {
        dp[i] = dp[i - 1] + dp[i - 2];
    }
    return dp[n];
}

Description: Items cannot be broken. A 2D table compares Items (rows) against Current Weight Capacity (columns) to find the absolute maximum value.

Time: $O(n \times W)$ Space: $O(n \times W)$
Java: 0/1 Knapsack (2D DP)
public int knapSack(int W, int[] wt, int[] val, int n) {
    int[][] dp = new int[n + 1][W + 1];
    for (int i = 1; i <= n; i++) {
        for (int w = 1; w <= W; w++) {
            if (wt[i - 1] <= w) {
                dp[i][w] = Math.max(val[i - 1] + 
                           dp[i - 1][w - wt[i - 1]], dp[i - 1][w]);
            } else {
                dp[i][w] = dp[i - 1][w];
            }
        }
    }
    return dp[n][W];
}

Description: Longest Common Subsequence. A 2D grid compares String A against String B to find the longest matching sequence of characters.

Time: $O(n \times m)$ Space: $O(n \times m)$
Java: LCS (2D DP)
public int lcs(String s1, String s2) {
    int m = s1.length(), n = s2.length();
    int[][] dp = new int[m + 1][n + 1];

    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (s1.charAt(i - 1) == s2.charAt(j - 1))
                dp[i][j] = dp[i - 1][j - 1] + 1; // Diagonal
            else
                dp[i][j] = Math.max(dp[i - 1][j], dp[i][j - 1]);
        }
    }
    return dp[m][n];
}
