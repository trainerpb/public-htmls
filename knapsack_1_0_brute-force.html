0/1 Knapsack: Brute Force

Before optimizing with Dynamic Programming, we must understand the Brute Force strategy. Since every item has a binary choice (0 = Leave, 1 = Take), a knapsack with N items has exactly 2N possible combinations.

There are two ways to program this: generating subsets via iteration (Bitmask), or building a decision tree via Recursion. Both have the exact same time complexity: O(2N).

public class KnapsackBruteForce {

    // 1. RECURSIVE APPROACH (The "Decision Tree")
    public static int knapsackRec(int W, int[] wt, int[] val, int n) {
        // Base Case: No items left or bag is full
        if (n == 0 || W == 0) return 0;

        // If item is heavier than remaining capacity, leave it
        if (wt[n - 1] > W) {
            return knapsackRec(W, wt, val, n - 1);
        } else {
            // Branch 1: Take It
            int takeIt = val[n - 1] + knapsackRec(W - wt[n - 1], wt, val, n - 1);
            // Branch 2: Leave It
            int leaveIt = knapsackRec(W, wt, val, n - 1);
            
            return Math.max(takeIt, leaveIt);
        }
    }

    // 2. ITERATIVE APPROACH (The "Bitmask")
    public static int knapsackIter(int W, int[] wt, int[] val, int n) {
        int maxProfit = 0;
        int totalCombinations = 1 << n; // 2^n

        // Loop through all subsets from 0 to (2^n - 1)
        for (int i = 0; i < totalCombinations; i++) {
            int currentWt = 0;
            int currentVal = 0;

            for (int j = 0; j < n; j++) {
                if ((i & (1 << j)) != 0) { // If j-th bit is 1, take item j
                    currentWt += wt[j];
                    currentVal += val[j];
                }
            }
            if (currentWt <= W && currentVal > maxProfit) {
                maxProfit = currentVal;
            }
        }
        return maxProfit;
    }
}

⏱️ Complexity: Recursive vs Iterative

Both methods shown are strictly Brute Force. They evaluate the exact same 2N possibilities.

  • Recursive Time Complexity: O(2N)
  • Iterative Time Complexity: O(N × 2N) (Because for each of the 2N combinations, we run a loop of size N to check bits).
  • Recursive Space Complexity: O(N) (Due to the system call stack going N frames deep).
  • Iterative Space Complexity: O(1) (No recursion stack, just a few integer variables).

🚨 The Fibonacci Trap! (Common Misconception)

Many students believe that merely changing a recursive function into an iterative while loop magically lowers the Time Complexity from O(2N) to O(N). This is completely false.

If you translate a recursive function into a loop that does the exact same logic (like our Bitmask above), the Time Complexity remains Exponential.

So why is the classic Iterative Fibonacci O(N)?
Because the standard iterative Fibonacci loop (c = a + b; a = b; b = c;) is NOT just "Iteration"—it is highly optimized Bottom-Up Dynamic Programming. It deliberately remembers previous answers to avoid redundant work. Iteration does not save time; remembering saves time.
