Problem 1: The Staircase Problem

The Setup: You are climbing a staircase that takes n steps to reach the top[cite: 177]. Each time you can either climb 1 step or 2 steps[cite: 178]. In how many distinct ways can you climb to the top? [cite: 179]

Algorithmic Paradigms

1. The Brute-Force Approach

Brute-force relies on exhaustive search[cite: 180]. For this problem, it means mapping out every single combination of 1-step and 2-step climbs from the bottom to the top[cite: 180].

  • How it works: You recursively branch out. At step 0, you branch into two parallel timelines: taking 1 step and taking 2 steps[cite: 181]. You repeat this exhaustive branching for every single position until you hit step n[cite: 182]. If a path perfectly lands on n, you count it as a valid way[cite: 183]. If it overshoots, you discard it[cite: 184].
  • Time Complexity: O(2^n) – The algorithm recalculates the exact same steps repeatedly, creating a massive, exponentially growing decision tree[cite: 184].
  • Space Complexity: O(n) – To maintain the recursion stack[cite: 185].

2. The Greedy Approach

Why Greedy Fails Here: The Greedy approach is entirely incompatible with this problem[cite: 188]. Greedy algorithms are designed exclusively for optimization (finding a single "best," "maximum," or "minimum" path based on local weights or values)[cite: 189]. The Staircase problem is a combinatorics problem—it asks for the total count of all possible paths[cite: 190]. Making a single "optimal" local choice (e.g., always taking 2 steps to get up faster) destroys the other branches of computation[cite: 191]. You cannot count exhaustive possibilities if you only ever pick one path[cite: 192].

3. The Dynamic Programming (DP) Approach

Dynamic Programming systematically solves and stores the answers to overlapping subproblems so that no calculation is ever performed twice[cite: 193].

  • How it works: To reach step n, you must have come from either step n-1 (by taking a 1-step jump) or step n-2 (by taking a 2-step jump)[cite: 194]. Therefore, the total number of ways to reach step n is simply the sum of the ways to reach n-1 and n-2[cite: 195].
  • The Logic: This boils down to the Fibonacci sequence: ways(n) = ways(n-1) + ways(n-2)[cite: 196]. Instead of recursively recalculating these paths like Brute-Force, DP computes the ways for step 1, then step 2, then step 3, storing each answer in an array (or just holding the last two values in memory) until it reaches n[cite: 197].
  • Time Complexity: O(n) – It only takes a single, linear pass to compute the answer[cite: 198].
  • Space Complexity: O(1) – If you only store the results of the previous two steps instead of a full array[cite: 199].

Generalizing to r Steps

If you expand the rules of the Staircase Problem to allow jumping up to 3 steps, or generalize it to r steps, the fundamental Dynamic Programming logic remains exactly the same, but the state calculation expands[cite: 203].

  • The Logic: The number of ways to reach step n is the summation of the ways to reach all r previous steps: ways(n) = ways(n-1) + ways(n-2) + ... + ways(n-r)[cite: 209].
  • Time Complexity: O(n × r)[cite: 209]. The DP algorithm must run a small loop of size r to sum up the previous r values for every single one of the n steps[cite: 210].
  • Space Complexity: O(r)[cite: 212]. Because the algorithm must constantly look back at the previous r steps, you need to maintain an array or sliding window of size r in memory to store those recent results[cite: 213].
