Activity Selection Algorithm

The Problem: You have a single resource (like a conference room) and a set of activities, each with a specific Start Time and End Time. The goal is to select the maximum possible number of non-overlapping activities.

The Greedy Strategy: Always pick the available activity that ends the earliest. By freeing up the resource as quickly as possible, you leave the maximum amount of time for future activities. Never look back.

import java.util.Arrays;
import java.util.Comparator;

public class ActivitySelection {
    public static void selectActivities(int[][] activities) {
        // Step 1: Sort activities based on their end time (column 1)
        Arrays.sort(activities, Comparator.comparingInt(a -> a[1]));

        System.out.println("Selected Activities:");
        
        // Step 2: The first activity is always selected
        int i = 0;
        System.out.print("(" + activities[i][0] + ", " + activities[i][1] + ") ");

        // Step 3: Consider the rest of the activities
        for (int j = 1; j < activities.length; j++) {
            // If this activity starts after or when the previous one finished
            if (activities[j][0] >= activities[i][1]) {
                System.out.print("(" + activities[j][0] + ", " + activities[j][1] + ") ");
                i = j; // Update the index of the last selected activity
            }
        }
    }
}

struct Activity { int start; int end; };

// Comparator for qsort to sort by end time
int compare(const void *a, const void *b) {
    return (((struct Activity *)a)->end - (((struct Activity *)b)->end));
}

void selectActivities(struct Activity arr[], int n) {
    // Step 1: Sort activities based on end time
    qsort(arr, n, sizeof(struct Activity), compare);

    printf("Selected Activities:\n");

    // Step 2: The first activity is always selected
    int i = 0;
    printf("(%d, %d) ", arr[i].start, arr[i].end);

    // Step 3: Consider the rest of the activities
    for (int j = 1; j < n; j++) {
        // If this activity starts after or when the previous one finished
        if (arr[j].start >= arr[i].end) {
            printf("(%d, %d) ", arr[j].start, arr[j].end);
            i = j; // Update last selected index
        }
    }
}

Time Complexity: O(N log N)

The time complexity is completely dominated by the sorting step.

  • Sorting the array of activities based on their end times requires O(N log N) operations.
  • Iterating through the array sequentially to make the greedy choices takes linear time, O(N).

Note: If the input data is already provided to you sorted by end-time, the algorithm operates in strictly O(N) time.

Space Complexity: O(1)

The Greedy strategy evaluates items in-place. We only need a few auxiliary variables (like storing the index of the last selected activity) to track our state. No large 2D arrays (like in Dynamic Programming) are required.

Note: Depending on the specific sorting algorithm implemented by the programming language (e.g., Merge Sort vs Quick Sort), the sorting function itself might utilize O(log N) or O(N) stack space.
