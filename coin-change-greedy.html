Greedy Algorithms

What does "Greedy" mean?

A greedy algorithm builds a solution piece by piece, always choosing the next piece that offers the most immediate benefit. It is called "greedy" because it acts on short-term gain. It never reconsiders its choices and never looks ahead to see if a current choice might cause problems later.

Local Solution (Optimum)

The choice that looks the absolute best right now, from your current position, without considering the rest of the problem.

Global Solution (Optimum)

The mathematically absolute best overall outcome for the entire problem from start to finish.

Requirements for a Greedy Algorithm

For a greedy approach to mathematically guarantee a Global Optimum, the problem MUST possess two specific properties:

  1. Greedy Choice Property: A global optimal solution can be arrived at by making a locally optimal choice. (i.e., making the greedy choice never locks you out of the best final answer).
  2. Optimal Substructure: An optimal overall solution to the problem contains optimal solutions to all of its sub-problems.

The Greedy Trap: Finding the Shortest Path Tree

We want to find the path with the lowest total cost to get from Start (Root) to End (Leaves).

Greedy Path (Left) = Total 92

At the Start, the Greedy algorithm looks at the immediate edges (Cost 2 vs Cost 5). It takes the local optimum (Cost 2). This traps it into a massive cost of 90 on the next jump!

Optimal Path (Right) = Total 9

To find the global optimum, the algorithm would have to sacrifice the immediate reward (taking the higher Cost 5 edge) to unlock the much cheaper Cost 4 edge later.

Complexity Analysis: While Loop vs Division Approach

As discussed in Lecture 21, solving the Coin Change problem greedily requires sorting the coins in descending order first, which takes O(n log n) time. After sorting, the algorithm iterates through the coins. There are two ways to do this:

Video Approach: The While Loop

Time Complexity: O(n · D) or O(n2) worst case.
Using a while(amount >= coin) loop manually subtracts the coin over and over. If the amount is massive (e.g. 1,000,000) and the coin is small (e.g. 1), the inner loop wastes time executing a million times.

Optimized Approach: Math (Division)

Time Complexity: O(n)
By replacing the while loop with standard mathematical division count = amount / coin and modulo amount = amount % coin, we determine the exact coin count in a single, constant O(1) step per denomination.

static void coinChange_DivisonApproach(Integer[] coins, int targetAmount) {
    Map<Integer, Integer> coinMap = new HashMap<>(); 
    
    /* Sort in reverse order : O(N*logN) */
    Arrays.sort(coins, Comparator.reverseOrder());
    System.out.println("Coins sorted (desc): " + Arrays.toString(coins));
    
    int countOfCoins = 0;
    
    for (int coin : coins) {
        if (targetAmount == 0) break;
        
        // Replaced while loop: handles it in 1 step!
        if (coin <= targetAmount) {
            int coinsTaken = targetAmount / coin;
            countOfCoins += coinsTaken;
            
            int thisCoinCount = coinMap.getOrDefault(coin, 0) + coinsTaken;
            coinMap.put(coin, thisCoinCount);
            
            targetAmount = targetAmount % coin;
        }
    }

    if (targetAmount > 0) {
        System.out.println("No solution. Remaining: " + targetAmount);
    } else {
        System.out.println(String.format("Total %d coins used. Map: %s", 
            countOfCoins, coinMap));
    }
}

static void coinChange_SubtractionApproach(Integer[] coins, int targetAmount) {
    Map<Integer, Integer> coinMap = new HashMap<>(); 
    
    /* Sort in reverse order : O(N*logN) */
    Arrays.sort(coins, Comparator.reverseOrder());
    System.out.println("Coins sorted (desc): " + Arrays.toString(coins));
    
    int countOfCoins = 0;
    
    for (int coin : coins) {
        if (targetAmount == 0) break;
        
        // Iterates based on amount size!
        while (coin <= targetAmount) {
            countOfCoins++;
            int thisCoinCount = coinMap.getOrDefault(coin, 0) + 1;
            coinMap.put(coin, thisCoinCount);
            targetAmount = targetAmount - coin;
        }
    }

    if (targetAmount > 0) {
        System.out.println("No solution. Remaining: " + targetAmount);
    } else {
        System.out.println(String.format("Total %d coins used. Map: %s", 
            countOfCoins, coinMap));
    }
}

def coin_change_division_approach(coins, target_amount):
    coin_map = {}
    
    # Sort in reverse order
    coins.sort(reverse=True)
    print(f"Coins sorted (desc): {coins}")
    
    count_of_coins = 0
    
    for coin in coins:
        if target_amount == 0:
            break
            
        # Constant O(1) step
        if coin <= target_amount:
            coins_taken = target_amount // coin
            count_of_coins += coins_taken
            
            coin_map[coin] = coin_map.get(coin, 0) + coins_taken
            target_amount = target_amount % coin
            
    if target_amount > 0:
        print(f"No solution. Remaining: {target_amount}")
    else:
        print(f"Used {count_of_coins} coins. Map: {coin_map}")

void coinChange_DivisionApproach(int coins[], int size, int targetAmount) {
    int coinCounts[size]; // Acts as our Map
    for(int i=0; i<size; i++) coinCounts[i] = 0;
    
    qsort(coins, size, sizeof(int), compare);
    
    int countOfCoins = 0;
    
    for (int i = 0; i < size; i++) {
        if (targetAmount == 0) break;
        
        // Handled in one math step!
        if (coins[i] <= targetAmount) {
            int coinsTaken = targetAmount / coins[i];
            countOfCoins += coinsTaken;
            coinCounts[i] += coinsTaken;
            targetAmount = targetAmount % coins[i];
        }
    }

    if (targetAmount > 0) {
        printf("No solution. Remaining amount: %d\n", targetAmount);
    } else {
        printf("We used total %d coins.\n", countOfCoins);
        // printCoinMap(coins, coinCounts, size);
    }
}
