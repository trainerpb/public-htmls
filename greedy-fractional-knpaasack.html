Fractional Knapsack Problem

Problem Statement

You are a thief carrying a knapsack that can hold a maximum total weight. You are in a vault with various items, each possessing a specific weight and a specific monetary value. Your goal is to maximize the total profit in your knapsack without tearing it.

✓ WHAT IS ALLOWED: Breaking items into smaller pieces! If an item weighs 5kg but you only have 2kg of space left, you are allowed to take a fraction (2/5) of the item and earn a fraction (2/5) of its value.

✗ WHAT IS NOT ALLOWED: Exceeding the maximum weight capacity of the knapsack.

The Three Greedy Strategies (Why Ratio Wins)

Let's assume our Knapsack Capacity is 6 kg. We have three items:
Item A (Value $100, Wt 5kg), Item B (Value $12, Wt 1kg), Item C (Value $75, Wt 3kg).

1. Greedy by Total Value

Strategy: Always pick the most expensive item first.
Action: Takes Item A ($100, 5kg). Leaves 1kg. Takes 1/3 of Item C ($25).
Total Profit: $125 (Fails because Item A is too heavy and hogs the bag).

2. Greedy by Min Weight

Strategy: Always pick the lightest item first to maximize item count.
Action: Takes Item B ($12, 1kg), then Item C ($75, 3kg). Leaves 2kg. Takes 2/5 of Item A ($40).
Total Profit: $127 (Fails because Item B is light but relatively worthless).

3. Greedy by Value/Weight Ratio

Strategy: Calculate "Bang-for-buck" (Value ÷ Weight). Sort descending.
Action: Item C ratio is $25/kg. Item A ratio is $20/kg. Takes all of C ($75, 3kg). Takes 3/5 of A ($60, 3kg).
Total Profit: $135 (Optimal!)

Complexity Analysis

Time Complexity: O(n log n)
Calculating the ratio for each item takes O(n). However, sorting the items based on their ratio takes O(n log n) time. The subsequent loop to fill the knapsack takes O(n). Therefore, sorting dominates the algorithm, giving an overall time complexity of O(n log n).

Space Complexity: O(1) or O(n)
If the sorting algorithm operates in-place and modifies the original array, it uses O(1) auxiliary space. If it requires allocating a new array or creating custom objects/structs to hold the ratios, it uses O(n) space.

import java.util.Arrays;
import java.util.Comparator;

class Item {
    int value, weight;
    public Item(int v, int w) {
        this.value = v;
        this.weight = w;
    }
}

public class FractionalKnapsack {
    public static double getMaxValue(Item[] items, int capacity) {
        // 1. Sort items by Value/Weight ratio in descending order O(n log n)
        Arrays.sort(items, new Comparator<Item>() {
            @Override
            public int compare(Item item1, Item item2) {
                double ratio1 = (double)item1.value / item1.weight;
                double ratio2 = (double)item2.value / item2.weight;
                return Double.compare(ratio2, ratio1); // Reverse order
            }
        });

        double totalValue = 0d;

        // 2. Greedily pick items O(n)
        for (Item item : items) {
            if (capacity - item.weight >= 0) {
                // Bag can hold full item
                capacity -= item.weight;
                totalValue += item.value;
            } else {
                // Bag is full after taking a fraction of this item
                double fraction = (double)capacity / item.weight;
                totalValue += (item.value * fraction);
                capacity = 0;
                break; // Knapsack is fully packed
            }
        }
        return totalValue;
    }
}

#include <stdio.h>
#include <stdlib.h>

struct Item {
    int value;
    int weight;
};

// Comparator for qsort to sort in descending order based on ratio
int compare(const void *a, const void *b) {
    struct Item *i1 = (struct Item *)a;
    struct Item *i2 = (struct Item *)b;
    double r1 = (double)i1->value / i1->weight;
    double r2 = (double)i2->value / i2->weight;
    
    if (r1 < r2) return 1;
    if (r1 > r2) return -1;
    return 0;
}

double fractionalKnapsack(struct Item items[], int n, int capacity) {
    // 1. Sort items O(n log n)
    qsort(items, n, sizeof(struct Item), compare);
    
    double totalValue = 0.0;
    
    // 2. Greedily pick items O(n)
    for (int i = 0; i < n; i++) {
        if (capacity - items[i].weight >= 0) {
            capacity -= items[i].weight;
            totalValue += items[i].value;
        } else {
            double fraction = (double)capacity / items[i].weight;
            totalValue += (items[i].value * fraction);
            capacity = 0;
            break;
        }
    }
    return totalValue;
}
